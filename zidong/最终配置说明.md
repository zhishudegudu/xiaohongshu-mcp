
```python

image_folder = "/app/images"

# 本地图片文件夹（Python 用于选择图片）
local_image_folder = r"E:\docker-xiaohongshu\images"
```

### 3. Publisher 初始化

```python
publisher = Publisher(

    config.image_folder,        # /app/images (发送给 MCP)
```


## 工作原理

### Docker 挂载配置 (docker-compose.yml)

```yaml
volumes:
  - ./images:/app/images
```

这意味着：
- 宿主机 `./images` 目录（相对于 docker-compose.yml）
- 映射到容器内 `/app/images`

### Python 代码逻辑

```python
# 1. 从本地文件夹选择图片
files = os.listdir(local_image_folder)  # E:\xiao\...\docker\images
selected = random.sample(files, count)

# 2. 构建容器路径
paths = [f"/app/images/{file}" for file in selected]

# 3. 发送给 MCP 服务器
mcp_client.publish_content(images=paths)
```

## 常见问题

### Q: 提示"找不到图片文件夹"？

**A**: 检查本地图片目录是否存在：
```bash
dir E:\docker-xiaohongshu\images
```

如果不存在，创建它：
```bash
mkdir E:\docker-xiaohongshu\images
```

### Q: 如何修改图片目录？

**A**: 修改 `config.py`：
```python
local_image_folder: str = r"E:\你的图片目录"
```

### Q: 如何切换到本地模式？

**A**: 设置环境变量：
```bash
$env:USE_DOCKER="false"
python example_simple.py
```

或修改 `config.py` 中的默认值：
```python
use_docker = os.getenv("USE_DOCKER", "false").lower() == "true"
```

### Q: 可以手动指定图片吗？

**A**: 可以！使用容器路径：
```python
images = [
    "/app/images/1.jpg",
    "/app/images/2.jpg",
    "/app/images/3.jpg"
]
publisher.publish(title, content, tags, images=images)
```
