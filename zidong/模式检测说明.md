# 模式检测说明

## 自动检测 Docker 模式

程序现在可以**自动检测**运行模式，无需手动设置环境变量！

## 检测逻辑

### 1. 优先级：环境变量
如果设置了 `USE_DOCKER` 环境变量，直接使用该值：
```bash
# 强制 Docker 模式
$env:USE_DOCKER="true"

# 强制本地模式
$env:USE_DOCKER="false"
```

### 2. 自动检测：连接测试
如果没有设置环境变量，程序会：
1. 尝试连接 `http://127.0.0.1:18060/api/v1/health`
2. 如果连接成功 → **Docker 模式**
3. 如果连接失败 → 默认为 **Docker 模式**

### 3. 默认值
如果无法确定，默认为 **Docker 模式**（因为大多数情况下使用 Docker）

## 检测函数

```python
def is_docker_mode() -> bool:
    """
    智能检测是否为 Docker 模式
    
    检测逻辑：
    1. 如果设置了环境变量 USE_DOCKER，使用该值
    2. 否则，通过检测 MCP 服务器响应头判断
    3. 默认为 Docker 模式
    """
    # 1. 优先使用环境变量
    use_docker_env = os.getenv("USE_DOCKER", "").lower()
    if use_docker_env == "true":
        return True
    elif use_docker_env == "false":
        return False
    
    # 2. 自动检测：尝试连接 MCP 服务器
    try:
        import requests
        response = requests.get(
            "http://127.0.0.1:18060/api/v1/health",
            timeout=2,
            proxies={"http": None, "https": None}
        )
        if response.status_code == 200:
            return True  # Docker 模式
    except:
        pass
    
    # 3. 默认为 Docker 模式
    return True
```

## 使用场景

### 场景 1: 正常使用（推荐）
**无需设置任何环境变量**，程序自动检测：
```bash
python example_simple.py
```

### 场景 2: 强制指定模式
如果自动检测不准确，手动指定：

**Docker 模式：**
```bash
$env:USE_DOCKER="true"
python example_simple.py
```

**本地模式：**
```bash
$env:USE_DOCKER="false"
python example_simple.py
```

### 场景 3: 临时切换
在同一个终端会话中切换模式：
```bash
# 使用 Docker 模式
$env:USE_DOCKER="true"
python example_simple.py

# 切换到本地模式
$env:USE_DOCKER="false"
python example_simple.py

# 清除环境变量，恢复自动检测
Remove-Item Env:USE_DOCKER
python example_simple.py
```

## 两种模式的区别

### Docker 模式
```
宿主机: E:\docker-xiaohongshu\images\
容器内: /app/images/
Python 发送: /app/images/xxx.jpg
```

**配置：**
```python
image_folder = "/app/images"
local_image_folder = r"E:\docker-xiaohongshu\images"
```

### 本地模式
```
本地: E:\新建文件夹\
Python 发送: E:\新建文件夹\xxx.jpg
```

**配置：**
```python
image_folder = r"E:\新建文件夹"
local_image_folder = r"E:\新建文件夹"
```

## 常见问题

### Q: 如何知道当前是什么模式？
**A**: 运行 `python 验证配置.py`，查看检测结果。

### Q: 自动检测不准确怎么办？
**A**: 手动设置环境变量：
```bash
$env:USE_DOCKER="true"  # 或 "false"
```

### Q: 为什么默认是 Docker 模式？
**A**: 因为大多数用户使用 Docker 运行 MCP 服务器，这样可以减少配置步骤。

### Q: 可以在代码中修改检测逻辑吗？
**A**: 可以！修改 `config.py` 中的 `is_docker_mode()` 函数。

### Q: 环境变量会持久化吗？
**A**: 不会。`$env:USE_DOCKER` 只在当前 PowerShell 会话有效。如果需要持久化，可以：
1. 添加到系统环境变量
2. 或在 `config.py` 中硬编码
