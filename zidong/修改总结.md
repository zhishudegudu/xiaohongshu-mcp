# 代码修改总结

## 问题原因

MCP 服务器运行在 Docker 容器中，只能访问容器内挂载的目录：
- **宿主机路径**: `E:\docker-xiaohongshu\images\`
- **容器内路径**: `/app/images/`

之前的代码使用了错误的路径（`E:\新建文件夹\`），导致容器无法访问图片。

## 修改内容

### 1. 修改 `config.py`

**修改前：**
```python
def get_default_image_folder() -> str:
    if platform.system() == "Windows":
        return r"E:\新建文件夹"  # ❌ 容器访问不到
```

**修改后：**
```python
def get_default_image_folder() -> str:
    """
    注意：如果 MCP 服务器运行在 Docker 容器中，
    需要使用容器内的路径，例如：/app/images
    """
    if platform.system() == "Windows":
        return "/app/images"  # ✅ 容器内路径
```

### 2. 简化 `publisher.py`

**删除的功能：**
- ❌ 图床上传功能（`image_uploader.py`）
- ❌ `use_image_host` 参数

**修改前：**
```python
def __init__(self, mcp_client: MCPClient, image_folder: str, use_image_host: bool = True):
    self.use_image_host = use_image_host
    
    if self.use_image_host:
        print("\n📤 上传图片到图床...")
        images = upload_local_images_to_urls(images)
```

**修改后：**
```python
def __init__(self, mcp_client: MCPClient, image_folder: str):
    """
    Args:
        image_folder: 图片文件夹路径（Docker 容器内路径，如 /app/images）
```

## 使用方法

### 1. 准备图片

将图片放到宿主机目录：
```
E:\docker-xiaohongshu\images\
├── 1.jpg
├── 2.jpg
└── 3.jpg
```

### 2. 运行程序

```bash
# 验证配置
python 验证配置.py

# 运行发布
python example_simple.py
```

程序会自动使用 `/app/images/` 路径（容器内路径）。

### 3. 手动指定图片

如果需要手动指定图片：
```python
images = [
    "/app/images/1.jpg",
    "/app/images/2.jpg",
    "/app/images/3.jpg"
]
publisher.publish(title, content, tags, images=images)
```

## 路径映射关系

| 宿主机路径 | 容器内路径 | Python 代码 |
|-----------|-----------|------------|
| `E:\docker-xiaohongshu\images\1.jpg` | `/app/images/1.jpg` | `/app/images/1.jpg` ✅ |
| `E:\新建文件夹\1.jpg` | ❌ 容器访问不到 | ❌ 不要使用 |

## 关键点

1. ✅ **图片放在**: `E:\docker-xiaohongshu\images\`
