# 代理配置说明

## 问题现象

运行程序时出现以下错误：
```
openai.APITimeoutError: Request timed out.
httpcore.ConnectTimeout: timed out
```

## 原因

在中国大陆访问 Gemini API 需要使用代理（梯子）。

## 解决方案

### 步骤 1: 确认代理端口

查看你的代理软件端口号：
- **Clash**: 通常是 `7890`
- **V2Ray**: 通常是 `1080` 或 `10808`
- **Shadowsocks**: 通常是 `1080`

### 步骤 2: 修改代码

编辑 `config.py` 文件，找到文件开头的代理配置部分（第 10-14 行）：

```python
# ================= 🔴 代理配置 =================
# 如果需要使用代理访问 Gemini API，取消下面两行的注释并修改端口
# os.environ["http_proxy"] = "http://127.0.0.1:1080"
# os.environ["https_proxy"] = "http://127.0.0.1:1080"
# ===============================================
```

**取消注释并修改为你的端口：**

```python
# ================= 🔴 代理配置 =================
# 如果需要使用代理访问 Gemini API，取消下面两行的注释并修改端口
os.environ["http_proxy"] = "http://127.0.0.1:1080"  # 改成你的端口
os.environ["https_proxy"] = "http://127.0.0.1:1080"  # 改成你的端口
# 重要：本地地址不走代理，避免影响 MCP 服务器连接
os.environ["no_proxy"] = "localhost,127.0.0.1,::1"
# ===============================================
```

**⚠️ 重要说明：**
`no_proxy` 配置确保本地 MCP 服务器（127.0.0.1:18060）的请求不走代理，只有 Gemini API 请求才使用代理。

### 步骤 3: 保存并运行

保存文件后重新运行程序：

```bash
python quick_start.py
```

## 示例配置

### 使用 Clash（端口 7890）

```python
os.environ["http_proxy"] = "http://127.0.0.1:7890"
os.environ["https_proxy"] = "http://127.0.0.1:7890"
```

### 使用 V2Ray（端口 1080）

```python
os.environ["http_proxy"] = "http://127.0.0.1:1080"
os.environ["https_proxy"] = "http://127.0.0.1:1080"
```

### 使用 V2Ray（端口 10808）

```python
os.environ["http_proxy"] = "http://127.0.0.1:10808"
os.environ["https_proxy"] = "http://127.0.0.1:10808"
```

## 验证配置

运行程序后，如果看到：

```
🧠 Gemini 正在创作主题：xxx
```

并且没有超时错误，说明代理配置成功！

## 注意事项

1. **确保代理软件正在运行**
2. **端口号必须正确**
3. **代理模式设置为全局或规则模式**
4. **如果还是超时，尝试增加超时时间**（在 `__init__` 方法中添加 `timeout=120.0`）
