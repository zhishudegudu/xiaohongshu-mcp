# 自动配图功能说明

## 功能介绍

程序现在支持**自动配图**功能！根据生成的内容，自动从 Unsplash 等免费图库下载合适的配图。

## 工作流程

```
1. AI 生成内容（标题 + 正文）
   ↓
2. AI 分析内容，生成图片搜索关键词
   ↓
3. 从 Unsplash 下载相关图片
   ↓
4. 使用下载的图片发布内容
```

## 使用方法

### 方法 1: 使用快速启动（推荐）

```bash
python quick_start.py
# 选择 4 - 自动配图示例
```

### 方法 2: 直接运行示例

```bash
python example_auto_images.py
```

### 方法 3: 在代码中使用

```python
from config import config
from mcp_client import MCPClient
from ai_generator import AIGenerator
from publisher import Publisher

# 初始化
mcp_client = MCPClient(config.mcp_server_url)
ai_generator = AIGenerator(
    api_key=config.gemini_api_key,
    base_url=config.gemini_base_url,
    model=config.gemini_model
)
publisher = Publisher(mcp_client, config.image_folder, config.local_image_folder)

# 1. 生成内容
result = ai_generator.generate_from_scratch("上海网红咖啡店")
title = result["title"]
content = result["content"]

# 2. 生成图片关键词
keywords = ai_generator.generate_image_keywords(title, content, count=3)
# 例如: ["coffee", "cafe", "latte"]

# 3. 下载配图
images = publisher.download_images_by_keywords(keywords, count=3)

# 4. 发布
publisher.publish(title, content, images=images)
```

## 常见问题

### Q: 下载的图片保存在哪里？

**A**: 保存在 `config.local_image_folder` 配置的目录中：
- Docker 模式：`E:\docker-xiaohongshu\images\`
- 本地模式：`E:\新建文件夹\`

文件名格式：`auto_{keyword}_{随机数}.jpg`

### Q: 下载失败怎么办？

**A**: 程序会自动回退到使用本地图片：
```python
images = publisher.download_images_by_keywords(keywords, count=3)
if not images:
    print("⚠️ 图片下载失败，使用本地图片")
    images = None  # 自动选择本地图片
```

### Q: 可以自定义关键词吗？

**A**: 可以！直接传入关键词列表：
```python
keywords = ["coffee", "latte", "cappuccino"]
images = publisher.download_images_by_keywords(keywords, count=3)
```

### Q: 需要 API Key 吗？

**A**: 不需要！Unsplash Source API 和 Lorem Picsum 都是免费的，无需注册。

### Q: 图片质量如何？

**A**: Unsplash 提供的都是专业摄影师拍摄的高质量图片，非常适合小红书风格。

### Q: 可以下载更多图片吗？

**A**: 可以，修改 `count` 参数：
```python
images = publisher.download_images_by_keywords(keywords, count=5)
```

### Q: 下载速度慢怎么办？

**A**: 
1. 检查网络连接
2. 使用代理（如果需要）
3. 减少下载数量
4. 使用本地图片作为备选

### Q: 图片会重复吗？

**A**: Unsplash Source API 每次返回随机图片，重复概率很低。文件名中包含随机数，避免覆盖。
